elk搭建社工库

发表于 2016-11-20   |   分类于 网络安全   |  
可以播放音乐哦！

利用ELK来搭社工库，ElasticSearch、Logstash、Kibana三个应用的缩写则是ELK, ElasticSearch简称ES，主要用来存储和检索数据。Logstash主要用来往ES中写入数据。Kibana主要用来展示数据。记录下笔记以免下次需要可以快速完成。

环境：Centos7
软件
elasticsearch-1.7.4.tar.gz jdk1.8 kibana-4.1.4-linux-x64.tar.gz logstash-2.1.1.tar.gz

JDK1.8安装

logstash

启动：

自动上传数据 这样就搭建好了。

而安装最新版的ElasticSearch有点不一样：
elasticsearch只能用非root启动，这里我创建了一个叫seven的用户
useradd seven passwd seven

修改配置文件：

这时候启动会报错的
修改：
sysctl -w vm.max_map_count=262144
或者永久性修改
/etc/sysctl.conf

上面还有一个错误是关于jvm内存分配的问题heap size [268435456] not equal to maximum heap size [2147483648]，需要修改的jvm配置

vim /usr/java/elasticsearch/config/jvm.options 将-Xmx2g改成-Xmx256m，也就是heap size [268435456] /1024/1024的值
config/elasticsearch.yml文件，找到如下配置，开放discovery.zen.ping.unicast.hosts及discovery.zen.minimum_master_nodes

然后修改max file descriptors [4096] for elasticsearch process likely too low, increase to at least [65536]这个错误（切换到root操作）
/etc/security/limits.conf seven hard nofile 65536
修改后重新登录seven用户，使用如下命令查看是否修改成功
ulimit -Hn

如果成功运行说明成功安装
