Information Shell Script

发表于 2016-10-22   |   分类于 Linux运维   |  
可以播放音乐哦！

这星期因别的事耽搁了，总共加起来写这脚本都没有一天时间，以后慢慢想到了继续添加

目录结构：

main.sh
这个程序开头写的比较渣，也难看还需要有所提高：

info.sh
这个主要用来调用错误信息，之前的想法是在这里写各种杂向其它函数信息来调用。主要显示错误之类的。

bin/nmap.sh
主要是经常使用nmap -sV -Pn IP -v -n，可能其它选项会忘记用，有些服务器禁止ping之类的，所以在渗透收集的时候最好是尝试多几次。不要因为一次扫描不到而不继续扫了。

bin/nmapvulns.sh
这个是加强班nmap

tools/subDomainBrute/domain_info.sh
经常用subdomain来爆破子域名，不过默认的话有4个字典都不相同，而我又不想漏掉这些文件，都执行了一次之后输出一个文件再进行全并去重处理，再之后通过bbscan来扫描敏感信息

tools/kali_system.sh
在kali下也有挺多工具的，所以就集成在一个脚本文件里，经常爆破端口的时候因为字典或者懒就不想去搞了。这里爆破写的太渣了，在写的过程中应该注释更清析，if语句不能套太多层了，自己看都晕了。

其实暂时还没有写完，只是为了完成每一周的博客日记，并不知道写什么，所以就把这些渣渣代码贴了上来，近期别的事情也很少学习 不过从现在起就有了那么多一点点学习时间了，还是需要利用时间碎片化这个太重要了，有时候我觉得这些碎片时间又不能怎么样，我的想法完全错了。
