BadUSB

发表于 2017-08-19   |   分类于 网络安全   |  

接到任务需要制作一款U盘植马工具，记得之前搞过Arduino Leonardo效果不是太理想，折腾的不算太久，在阅读各种相关文章后自己也学到了很多，此次先是同事买了一个SanDiskU盘，因为参考连接是支持这种的：Known Supported Devices，以及还有：Make your own USB Rubber Ducky using a normal USB stick
GitHub Psychson Link
搞了一天才发现这种板子根本就不能支持。
这里有个制作视频很详细：BadUSB - HID Attack

后来买了这个：

880个人认为觉得不值，还好花的不是我的钱。

  • 1.开发板，带了一个128M内存卡
  • 2.读卡器

cve-2017-8464

这是一个微软Windows系统处理LNK文件过程中发生的远程代码执行漏洞。
当存在漏洞的电脑被插上包含漏洞利用工具的U盘时，不需要任何额外操作，漏洞攻击程序就可以借此完全控制用户的电脑系统（该漏洞也可能由用户访问网络共享、从互联网下载、拷贝文件等操作被触发和利用攻击）。

https://github.com/rapid7/metasploit-framework/tree/master/data/exploits/cve-2017-8464
https://github.com/rapid7/metasploit-framework/blob/master/modules/exploits/windows/fileformat/cve_2017_8464_lnk_rce.rb
下载到对应的地方。MAC安装后的默认路么：
/opt/metasploit-framework/embedded/framework/modules/exploits/windows/fileformat

其它系统默认路径：
/opt/metasploit/apps/pro/vendor/bundle/ruby/2.3.0/gems/metasploit-framework-4.14.8/modules/

将生成的文件全部复制进U盘。
