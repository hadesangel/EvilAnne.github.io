一星期实战总结（二）

发表于 2017-03-11   |   分类于 网络安全   |  

struts后的一些想法

最近这段时间爆出来非常严重的漏洞Struts2远程命令执行,然后我看到了一群人都在刷，提交到漏洞平台上，还有某些团队一年都搞不定的站，现在都可以拿下了，现在来说点我的想法，第一种人我觉得最无聊之人到处刷各种刷，然后可以在某些平台上排上高的名次。然而第二种人是已经工作了，是有目的去搞这是他们的一个绝加好机会，漏洞刚爆出来去搞了很多目标站，相当刚爆出来的洞搞下了，总比自己要花上数月时间去搞定目标站。
然而我想难道是不是一直要跟着别人后面走，利用别人的漏洞去搞站，现在想想自己所学的东西都是前人的，包括一些漏洞信息泄漏都是前人分享出来，自己才知道有这样的技巧。
我总觉得我们上面有人，觉得下面已经没什么漏洞爆发出来了，所以就把一个用到不用的东西发出来给这个圈子点活力，让那些人去完成很久搞不下的东西。
有时候无知也是一种福，不需要去看透这个世界，然而我并非说自己已经看透了这个世界。只是觉得这个圈子人外有人，山外还有一山，不管你多厉害，总有一个比你还厉害。

这里还需要装几个包multipart_encode、register_openers,需要在当前目录下创建一个空文本tmp.txt，使用方法：python struts.py http://localhost/index.action，这里程序已经定义了如果存在则执行ifconfig，再最初通过这个命令拿下之前遇到的struts，但在添加用户的时候又遇到了问题，尝试了很多次发现是这poc的问题，所以写法要点不一样，useradd ccav 'echo '123456'|passwd --stdin ccav',当时echo "123456"一直用双引号。添加不成功 ，试多几次就成功了。给个案例给的的经验的，渗透就是要多尝试，坚持、说不定下一次回车键就是奇迹的发生。

cat /etc/sysconfig/network-scripts/ifcfg-eth0:
ip是172.开头属于B类网络地址。查看DNS配置文件是在内网中的dmz区域，无法上网。
而通过whoami权限也是最小权限,这老外真是牛B，因为一直通过struts脚本去执行一些命令不怎么好，先获取WebShell，搞了很长一段时间都没有解决。大概搞了有一下午到晚上9点才找到路径，通过find命令查找也得不到，有也只是一些备份文件。完全没有JSP的文件。
然后分析服务器是JBOSS通过df -h查看磁盘，发现jboss放在某一处。最后一个一个翻，翻到某个domain.war文件下才看到了源码文件，也写了一个文件正常显示。
好吧，经验不足下次直接通过这条命令来查找find / -name "*.war"。
直接写JSP一句话，木马，不过写进去之后500无法连接，过一会自动删除。

写进去。通过k8来传马。后来发现目录下有人已经上传了 WebShell、查看了他的JSP小马是国外。小马还加了密码。这里不就分享出来。
可能看似一个简单的过程，里面还有很多细节和想法都没有写出来。
因为权限也是太小，所以找到Oracle的密码，通过Socks5代理出来，发现代理没有流量是因为这台机器根本就没有办法访问互联网。
这时候又派了另外一个任务过来，所以这站暂时就放了2天。 等星期五再去看的时候已经被修复了，马也被删了。有点心疼。
这告诉我，以后搞出点什么，找到个口子，不管怎么多大的时间代价也要去搞。

接到目标一个网址、三个ip，这3个IP为阿里云服务器、这里暂时略过、先分析站点。
1.目标站类型：类似一个投资站点
2.程序：Wordpress

看到Wordpress就觉得有点蛋疼，不过没有放弃继续看,这里称为A站，先通过wpscan没找出什么，再通过wpscan把网站的管理员人员爬出来，再通过BurpSuite爆破，利用常见弱口令密码可想而之以失败造终。然后通过F12发现会员登录页面其实是另外一个站点的这里我称为B站,类似一个赌博网站、发现会员登录页面又跳转到另外一个金融投资的站点，这里我称为C站，从中午一直到下午5点将近就收集了点信息。看到是很难搞，也不好搞呀。站在窗边看着远方…
回到电脑前，整理一下思路，重新信息收集，发现这个站点的后台，只要一个登录页面。不过好在运气好发现svn泄漏把整个网站源码下回来。看了一下是国内某个框架比较老的版本。现在思路是通过代码审计找到网站漏洞，此时已经发现了内网多台数据库连接的用户名和密码。
第二天早的比较早，毕竟是这个星期的最后一天，希望能搞出点什么来。
先是从头开始分析代码之前先读了index.php文件一直到config配置文件，看着看着就尝试打开XSHELL去尝试连接，发现从所得到的信息，再猜测root密码，后来经过分析，只要有关联的服务器，只要得到一个域名，大概可以猜测出密码的规律。然后把Wordpress密码脱了出来，不过密码不可解就先放着了。再到后来因为网络比较大，域名比较多，需要拿到核心的数据。经过一个下午获取所有东西。
这里其实并没有深入内网里面，毕竟每操作多一步，都会记录在服务器中。

写这个日志只是为了大概记一写，写不了太详细。毕竟是见不得人的内容。
有时候也在想，做到这一步是否适可而止，还是继续深入去研究。曾经刚学的时候入侵了学校网站当时只是拿到了WEB服务器，其实还连接于内网，那时候觉得很有成就感没有继续往下搞。

总结

一个星期、5天，自己为自己努力，做不出来的去想，去思考。总比放弃要强得多。去试去做，总比迷茫如何走哪条路强得多。
